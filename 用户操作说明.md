# TwinFlow 用户操作说明

本文档用于指导用户使用 `TwinFlow` 数字孪生页面，包括 3D 场景浏览、蓝图配置、MQTT 接入和常见问题排查。

## 1. 页面组成

页面分为三个主要区域：

- 顶部标题区
  - 显示系统名称和当前选中设备
- 上方内容区
  - 左侧是 `3D 场景预览`
  - 右侧是 `运行控制` 与 `设备配置`
- 下方内容区
  - 是完整的 `MQTT Blueprint 工作台`

## 2. 3D 场景操作

- 鼠标左键拖动：旋转视角
- 鼠标滚轮：缩放场景
- 鼠标右键拖动：平移视角
- 点击设备模型：选中该设备，右侧 `设备配置` 面板将显示该设备属性

选中设备后：

- 顶部会显示当前选中的设备 ID
- 在蓝图中配置“更新 3D 设备”节点时，可直接使用当前选中设备

## 3. 设备配置 (New)

选中设备后，可以在右侧面板进行以下操作：

- **模型自定义**: 输入 `.glb` 或 `.gltf` 链接即可替换默认方块模型。
- **变换调整**: 精确调整位置 (Position)、旋转 (Rotation) 和缩放 (Scale)。
- **实时同步**: 修改后 3D 场景立即更新。

## 4. 运行控制区

运行控制区主要用于连接 MQTT 和注入测试数据。

### MQTT WebSocket 地址

用于填写 MQTT Broker 的 WebSocket 地址，例如：

```text
ws://broker.emqx.io:8083/mqtt
```

### 模拟注入 Topic

用于指定手动模拟消息注入到蓝图时所使用的 Topic，例如：

```text
devices/pump-001/status
```

### 按钮说明

- `连接 MQTT`
  - 连接到当前填写的 MQTT Broker
- `断开 MQTT`
  - 断开 MQTT 连接
- `运行蓝图`
  - 让蓝图进入运行状态
- `停止蓝图`
  - 停止蓝图运行
- `注入到蓝图`
  - 使用当前选中的模拟消息，通过指定 Topic 注入蓝图

### 状态说明

- `Connected`
  - MQTT 已连接
- `Offline`
  - MQTT 未连接
- `Blueprint Running`
  - 蓝图处于运行状态
- `Blueprint Idle`
  - 蓝图处于停止状态

### 支持的节点

- `MQTT Subscribe`
- `MQTT Publish`
- `Inject`
- `Filter`
- `Map`
- `Update 3D Device` (现在支持 rotation 和 scale)
- `Timer` (New: 支持 Cron 表达式和间隔触发)
- `Delay` (New: 延迟执行)
- `Switch` (New: 多路分支路由)
- `Logger`

## 5. 蓝图工作台操作

蓝图工作台用于配置消息处理逻辑。

### 4.1 添加节点

可以通过两种方式添加节点：

- 从左侧节点面板拖拽到画布
- 直接点击节点卡片，自动添加到画布中

### 4.2 连线

- 从一个节点右侧输出端拖出连线
- 连接到另一个节点左侧输入端

### 4.3 编辑节点属性

- 点击画布中的节点
- 右侧“属性面板”会显示该节点配置项
- 修改后立即生效

### 4.4 工具栏按钮

- `运行`
  - 运行蓝图
- `停止`
  - 停止蓝图
- `校验`
  - 检查蓝图是否存在循环依赖或缺少必填配置
- `保存到本地`
  - 保存当前蓝图到浏览器本地存储
- `读取本地`
  - 从浏览器本地存储读取蓝图
- `导出 JSON`
  - 将当前蓝图导出为 JSON 文件
- `导入 JSON`
  - 从 JSON 文件恢复蓝图
- `清空日志`
  - 清除运行日志
- `清空画布`
  - 删除当前画布中的所有节点和连线

## 6. 节点说明

### MQTT 订阅

作用：

- 订阅一个 MQTT Topic
- 当 Topic 收到消息时，触发下游流程

主要配置：

- `Topic`
- `QoS`

### MQTT 发布

作用：

- 根据上游消息发布新的 MQTT 消息

主要配置：

- `Topic 模板`
- `Payload 模板`
- `QoS`
- `Retain`

### 手动注入

作用：

- 点击运行后，作为测试数据输入源

主要配置：

- `Payload JSON`

### 条件过滤

作用：

- 按条件判断是否继续向下游传递消息

主要配置：

- `字段路径`
- `操作符`
- `比较值`

示例：

```text
字段路径: payload.temperature
操作符: >=
比较值: 80
```

### 数据映射

作用：

- 把输入消息转换成新的 payload 对象

主要配置：

- `映射 JSON`

示例：

```json
{
  "id": "payload.id",
  "temperature": "payload.temperature",
  "status": "payload.status",
  "color": "payload.color"
}
```

### 更新 3D 设备

作用：

- 更新 3D 场景中的设备状态

支持的属性：

- `status`
- `temperature`
- `color`
- `position`
- `rotation`
- `scale`

主要配置：

- `设备`
- `属性`
- `值路径`

### Timer (New)

作用：

- 支持 Cron 表达式和间隔触发

### Delay (New)

作用：

- 延迟执行

### Switch (New)

作用：

- 多路分支路由

### 日志输出

作用：

- 把当前节点输入输出写入下方日志面板

## 7. 推荐操作流程

建议按照下面顺序进行：

1. 先点击 3D 场景中的设备，确定目标设备
2. 在蓝图画布中添加节点
3. 配置节点属性
4. 用连线把节点连起来
5. 点击 `校验`
6. 点击 `运行`
7. 用 `注入到蓝图` 或真实 MQTT 消息测试
8. 查看 3D 场景和运行日志

## 8. 示例流程

### 示例一：手动注入高温告警

流程：

```text
手动注入 -> 数据映射 -> 更新 3D 设备 -> 日志输出
```

效果：

- 更新设备温度
- 更新设备状态
- 更新设备颜色
- 在日志中查看输入输出

### 示例二：MQTT 订阅后自动更新设备

流程：

```text
MQTT 订阅 -> 条件过滤 -> 更新 3D 设备 -> MQTT 发布
```

效果：

- 收到设备状态消息后自动处理
- 当温度超过阈值时更新设备颜色
- 同时发布确认消息

## 9. 常见问题

### 1. 蓝图没有反应

排查顺序：

- 是否点击了 `运行`
- 节点之间是否已经正确连线
- 是否有校验错误
- `MQTT 订阅` 的 Topic 是否和实际消息一致
- 日志面板中是否有错误信息

### 2. MQTT 已连接，但没触发

排查顺序：

- 蓝图是否处于运行状态
- Topic 是否匹配
- 是否使用了通配符但格式不正确
- 消息内容是否为合法 JSON 或预期格式

### 3. 设备没有更新

排查顺序：

- 是否已选中正确设备
- “更新 3D 设备”节点中的 `设备` 是否配置正确
- `属性` 是否在支持范围内
- `值路径` 是否能从消息中取到值

### 4. 导入蓝图失败

排查顺序：

- JSON 文件是否被手动修改坏了格式
- `nodes`、`edges`、`viewport`、`metadata` 字段是否存在

## 10. 当前版本说明

当前版本是 MVP，特点如下：

- 前端纯 JS 执行引擎
- 不依赖后端规则服务
- 适合原型验证、演示和本地联调

当前暂不包含：

- 真实 GLB 动画控制
- 定时器 / Cron 节点
- 自定义 JS Function 节点
- 后端权限与历史数据管理
